🤖 AI sidebar context for this modal: page.id=preview-apply-modal action=remove_ac+log_incident visible_data=[diff, scope impact, scope limits] · suggested: /cross-check AC-5 first Show similar past incidents What does undo do exactly?
  1. Header ← action.name + action.is_destructive (boolean drives red styling). Closing × triggers Cancel without state mutation.
  2. Source-of-action callout ← action.source_type=ai_suggestion + action.source_conversation_id (sidebar). For user-initiated actions, swaps to "you initiated this from {page}".
  3. Diff (current vs proposed) ← computed by /action/preview: serializes current entity state, applies the action's mutation in a tx, returns serialized proposed state. No actual mutation persists. Highlighted lines = JSON-patch diff result.
  4. Scope of change ← action.affected_entities[] declared by the action handler. Each entity has type, id, change_kind, summary. UI groups by + (created), − (removed), · (unchanged-but-counter-affected).
  5. Scrutiny — "does NOT do" ← action.explicit_scope_limits[]. Required by action contract. Removing this list is a template violation. Each bullet links to the related action user could run instead.
  6. Apply / Save as draft / Cancel — Apply → POST /action/{type}/execute with idempotency key, redirect back with flash + audit log entry. Save as draft → POST /action/{type}/draft, draft visible on user's home + on entity. Cancel → close modal, no state.
  7. Reversibility note ← action.is_reversible + action.undo_window_hours. If non-reversible, the note becomes "PERMANENT" in red.
