SKILL Load test probe SK-load-probe

After any develop task, runs a lightweight load test against touched endpoints and reports p95/p99 latency.

applies_to_phases: develop, challenger · tags: performance, migration

Optional expression. Leave empty to fire on every stage completion. Expression runs against the completed task's context.

Available functions: diff_touches(), task.type, objective.scope, objective.has_ac(), task.duration_s. Full reference →
By default, hooks don't fire on replays (to avoid double-cost for historical audits). Enable only if the hook's output is itself version-dependent.

🔎 Things this hook will NOT do
  • Will not run before the stage — this is a post-stage hook. If you want pre-stage gates, use AC + task's "required_before_start" field.
  • Will not detect issues that take longer than the hook timeout (default 60s). Long-running load tests must be scheduled as their own tasks.
  • Will not pass state to the next hook in the chain — each hook fires with fresh context. Use skill's context API if you need state.
  • Will not retry on transient failures by default. Enable via advanced options (not in this form yet).
  • Your condition diff_touches(['app/api/**', 'app/cdc/**']) will not match tasks that only touch tests or migrations, even if they logically belong to CDC work. Broaden if needed.

12-add-hook · data sources

  1. Stage picker ← static enum. Stores hook.stage.
  2. Skill picker ← skill WHERE project_id=acme-erp-migration OR is_org_level=true. Grouped by skill.category. Preview pulls skill.description + skill.applies_to_phases + tags.
  3. Condition ← hook.condition_expression (TEXT). Server-side validator parses expression, rejects unknown functions/fields.
  4. On failure ← hook.on_failure enum [raise_finding, block_stage, warn_only].
  5. Cost estimate computed client-side: skill.cost_per_call × estimated_firings_per_month. Estimated firings = historical stage-completion rate × (condition match probability, default 0.7).
  6. Scrutiny partly static + partly condition-aware (the last bullet analyzes the user's condition expression for common gotchas).
