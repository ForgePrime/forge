1. SCOPE

Phase 1 covers migration of the order-to-cash pipeline from the legacy Oracle DB to a new PostgreSQL 16 cluster. The legacy Oracle DB must not be shared beyond read-only access. Migrate tables in batches; use CDC for the hot-path (orders, invoices) and nightly ETL for reporting.

The new cluster will be deployed on-prem in the existing data centre (rack DC1-R12). It will be sized to handle 2× current peak load — see Appendix B for benchmarks.

Out of scope for Phase 1: master data services migration, BI/reporting tier (separate workstream), mobile clients (existing apps continue to read via REST gateway).
