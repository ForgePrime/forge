Challenger-check tradeoffs you should know

  • More checks ≠ more safety. Opus attention is finite. The 3rd+ check dilutes focus; the challenger starts writing "applies: yes, evidence: see above" for everything.
  • Mandatory = blocking. If the challenger can't find evidence, the task cannot hit DONE. A poorly-worded mandatory check will block legitimate work.
  • Advisory is almost free to keep, but almost free to ignore. Nits get captured in review, rarely acted upon. Use sparingly.
  • Checks are NOT tests. A challenger can miss evidence that exists. Mandatory checks that NEED testing should have a companion Gate (see hooks tab).
  • Scenarios vs checks vs AC. Scenarios describe behavior. AC is what verifies behavior. Checks are instructions to the challenger about what to look at. Don't duplicate across all three.
  • Removing a check does NOT delete past challenger reports that cited it — those stay in task deliverables for audit.

09-edit-challenger-checks · data sources

  1. Header ← objective.{id, title} + count of objective.challenger_checks[].
  2. Check rows ← objective.challenger_checks[] JSONB. Each check: {id, text, severity, applies_to_task_types[], source: 'user'|'scenario'|'org'}. Firing stats ← aggregation from challenger_firing WHERE objective_id=O-002 GROUP BY check_id.
  3. Add new check → PATCH /objective/{id}/challenger-checks appending new entry. "🪄 Suggest from scenarios" opens the indigo panel below, which queries scenario WHERE objective_id=O-002.
  4. Suggested from scenarios ← LLM-free: pattern-matches scenario's expected_behavior into a "Confirm ... Require evidence ..." template. User checks which to promote → batch PATCH.
  5. Prompt injection preview ← client-side render of how objective.challenger_checks[] serialize into the Phase C prompt template. Deterministic — no LLM call to render.
  6. Cost display ← computed: sum(token length of each check text) × avg challenger invocation cost per token. Updates live as user edits.
  7. Scrutiny — challenger tradeoffs ← enforced template. Always visible. Removing is a template violation.
