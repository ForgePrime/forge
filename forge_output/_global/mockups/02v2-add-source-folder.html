🔎 What Forge did NOT check

  • File-content PII (SSN, credit-card, API-keys) — no regex scan on parse
  • License headers — GPL-contaminated code indexed without warning
  • Binary file detection beyond extension — a .md containing base64 will still embed
  • Symlink loops — we don't follow symlinks by default, but a loop in real paths would still recurse
  • File ownership / permissions audit — any file you can read, we index

02v2-add-source-folder · data sources

  1. Folder path ← source.path. Test → POST /source/folder/test returns {accessible, file_count, total_bytes, fs_type, user, group}. Server enforces project.folder_sandbox_root — paths outside it are rejected.
  2. Scan behavior flags ← source.recursive, source.follow_symlinks, source.respect_gitignore.
  3. Globs ← source.include_patterns[], source.exclude_patterns[]. Default sets provided as defaults in UI but stored verbatim. Match uses double-star globs (gitignore semantics).
  4. Sample scan preview ← synchronous POST /source/folder/preview applies globs, returns first 10 paths + count + token estimate. No embeddings produced.
  5. Description / focus / scopes / schedule ← source.description, source.focus_hint, source.scopes[], source.rescan_cadence. Same shape as file/URL sources.
  6. Sensitive-file warning ← enforced template. Shown whenever globs don't exclude .env*, *.key, *.pem. Inline suggestions derived from folder contents — if scanner found an .env, that suggestion is highlighted red.
Destinations: "Add + first scan" → POST /source then /source/{id}/scan. "Add without scan" → just the first POST. Redirect 02v2-project-kb.html?flash=SRC-008-{scanning|registered}.
