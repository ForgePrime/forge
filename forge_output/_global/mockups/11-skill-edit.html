11-skill-edit · data sources

  1. Name/ext ID/category/description ← skill row columns. Ext ID is primary lookup key — validator rejects changes if other entities reference it.
  2. Prompt text ← skill.prompt_template TEXT. Variable substitution happens at task-execution time via Jinja-style replacement.
  3. Applies_to_phases ← skill.applies_to_phases[] enum array. Enforced at runtime — skill won't attach outside these phases.
  4. Tags ← skill.tags[] JSONB array. Used for search/filter in library.
  5. Auto-attach rule ← skill.auto_attach_rule JSONB. Visual builder + JSON editor are two views of the same data. Saved rule is validated server-side (schema check).
  6. Cost impact ← skill.cost_impact_usd (user estimate) + llm_call aggregations (observed avg/p95) shown for comparison.
  7. Conflict check ← server-side: given new rule, simulate against task WHERE status=DONE AND created_at > NOW() - 30d across all projects using this skill. Shows diff. Comes from endpoint /skill/{id}/preview-rule-change.
Destinations: Save + publish → POST /skill/{id}, writes skill_version row, updates skill.current_version. Redirect 11-skills-library.html#SK-security-owasp. Save as draft → skill_version.status=DRAFT, no publish. Cancel → discard, redirect back.
