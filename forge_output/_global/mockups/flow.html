Invariants enforced by this flow

  1. Project ≠ SOW. Project is an initiative. KB is built over time.
  2. Objectives are where work lives. Analysis, planning, develop, docs all happen on an objective — not globally.
  3. Ambiguity at any stage bubbles up to its objective. Analysis flags it · planning can flag new ones · develop can too.
  4. Re-analysis is manual. After user resolves ambiguities, Forge waits for explicit "Continue with my answers" — no auto-trigger.
  5. Failure splits two ways: LLM self-fixable (retry) OR needs user decision (back to objective as new Q).
  6. Docs have two layers: auto-draft (deterministic, free) + DOCUMENTATION task (LLM-polished).
  7. ACHIEVED is reversible. User adds gap notes → objective re-opens → history preserved → only what's needed re-runs.
