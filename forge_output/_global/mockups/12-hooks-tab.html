Project configuration

Hooks fire AFTER a stage completes, regardless of individual task config. Used for project-wide invariants (e.g., "always re-check security after any analysis update").

Hooks run unconditionally after their stage. Unlike skills on tasks, you can't selectively skip a hook per task — if you need conditional firing, use the "only fire if..." condition on the hook itself.
🔎 What hooks do NOT do
  • Hooks do not run before a task — that's a pre-stage gate, not supported here. Use AC + verification instead.
  • Hooks do not block stage completion by default. A hook that fails only raises a finding; the task still completes. Escalate via hook's "on failure" setting.
  • Hooks do not share state across runs. Each hook fires with fresh context — if you need stateful checks, store output via skill's context API.
  • Hooks do not run on replays by default (to avoid double-cost). Opt-in via hook's "run on replay" toggle.
