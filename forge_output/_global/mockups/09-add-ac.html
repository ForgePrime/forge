09-add-ac · data sources

  1. Header ← objective.id/title + COUNT(ac WHERE objective_id=O-002). Next AC id reserved at form-mount (AC-6 shown in preview).
  2. 🪄 Regenerate → POST /ac/regenerate with {draft_text, objective_id, attached_sources[]}. Overwrites textarea with Claude output. Cost from skill.cost_impact.
  3. AC text ← ac.text. Server-side validation: ≥20 chars, no trailing "TBD", not duplicate of existing AC on same objective (fuzzy match).
  4. Scenario type ← ac.scenario_type enum [positive|negative|edge_case|regression].
  5. Verification ← ac.verification_method enum [test|command|manual]. Conditional test_path/command field is required iff method matches. Manual → generates manual_verification_task at completion.
  6. Source attribution ← ac.source_ref (FK to source.id) OR ac.source_ref='INVENTED' boolean. "INVENTED" triggers ac.is_unsourced=true which drives the red banner + scrutiny-debt counter. Passage quote ← ac.source_passage (TEXT, 2000 chars max).
  7. Preview + validation = client-side computed. No server roundtrip until Save.
Destinations: "Save AC" → POST /objective/O-002/ac → redirect 09-objective-detail.html#ac-6. "Save as draft" → ac.status=DRAFT (not surfaced to challenger). Cancel → 09-objective-detail.html.
