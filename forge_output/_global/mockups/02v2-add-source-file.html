02v2-add-source-file · data sources

  1. Drop zone + selected file ← multipart upload → POST /source/file/upload → returns upload_token + parse metadata (file.mime, file.bytes, file.page_count). No source row yet.
  2. Description (required) ← source.description (min 30 chars, server-validated). LLM suggest → POST /source/suggest-description using first 2k chars of parsed text.
  3. Focus hint ← source.focus_hint. Free text, indexed for chunker prompt.
  4. Scopes ← source.scopes[]. Tag chips; autocomplete from project.known_scopes[]. Used as filter in objective.kb_scope[].
  5. Extraction preview ← source.extracted_text_preview (first 400 chars stored after parse). Chunks are generated at commit, not at preview.
  6. Conflict detection ← /source/overlap-check against existing source_chunks using embedding similarity + term overlap. Returns list of {src_id, chunk_id, overlap_score, snippet}. Only score > 0.82 shown.
  7. Scrutiny counterweight ← enforced template. Lists what ingestion did NOT do: quality check, language detection, author verification, PII scan, license audit.
Destinations: "Add + queue" → POST /source w/ queue_for_analysis=true, redirect 02v2-project-kb.html?flash=SRC-006-added. "Add only" → same POST w/o queue. Cancel → discard upload_token, back to KB.
