🔎 Auth scope narrowing

Forge can only read; it cannot modify remote docs. The OAuth token requests:

  • Sites.Read.All — read doc lists
  • Files.Read — read file contents

NOT requested: Files.ReadWrite, Sites.Manage. You cannot accidentally grant write scope.

⚠ Stale-content warning

02v2-add-source-url · data sources

  1. URL + test connect ← source.url. Test → POST /source/url/test returns HEAD probe result. No auth used here (so public probes only).
  2. Auth type ← source.auth_kind enum [none|basic|token|sharepoint_oauth|confluence_oauth]. SharePoint auto-detected from hostname match against *.sharepoint.com. OAuth token stored in source.credentials (encrypted at rest, never logged).
  3. Crawl scope ← source.crawl.kind [single|recursive|same_domain] + source.crawl.max_depth + source.crawl.include_patterns[] + source.crawl.exclude_patterns[] (globs).
  4. Schedule ← source.recrawl_cadence enum [manual|daily|weekly]. Stored as cron expression internally. Cost estimate uses avg_diff_bytes_per_crawl × cron frequency.
  5. Crawl preview ← synchronous dry-run /source/url/dry-run returns first N URLs that pass include/exclude + robots.txt. Uses auth token from form state. No embeddings produced.
  6. Auth scope narrowing ← OAuth app scope declared in providers.sharepoint.scopes_requested (read-only). Enforced at IdP level — cannot be overridden from UI.
  7. Stale-content warning ← always shown when source.recrawl_cadence !== 'webhook'. Not optional.
