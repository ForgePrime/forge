🏗 Architecture overview DOC task

Produced by DOC-001 on O-002 · 2h ago · $0.42

The migration uses a strangler-fig pattern over 4 months. New code runs alongside the legacy ERP; data is synchronized via change data capture (CDC) from the legacy PostgreSQL into the new schema.

The system is on-prem only (constraint from client manual note SRC-004). Services are containerized for reproducibility but deployed inside the customer firewall.

Auth is session cookie-based with HIPAA-grade audit log — every read/write of a patient record is recorded. SSO was deliberately deferred to Phase 2 (decision D-001).
