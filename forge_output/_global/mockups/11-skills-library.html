11. Skills library — annotations

  1. Three categories, not one. SKILL (broad) / MICRO (narrow, deterministic) / OPINION (persona-based). Each has distinct mental model — don't collapse.
  2. Cost impact surfaced up-front. Skills add context → cost. The list is not "more is better" — it's a budget tradeoff.
  3. Auto-attach rules. LLM decides for broad skills (e.g., "if diff touches auth → attach security skill"). User can edit rule. Default set runs if nothing picked.
  4. Conflict warnings. "Cloud architect" opinion vs SRC-004 on-prem constraint → flag before install. Skills should not silently undermine decided constraints.
  5. Default skills per phase. If user picks nothing, these run. Editable at project level (→ next mockup 12). Prevents "blank slate" paralysis.
End goal: Forge autonomous. User only edits skill set during onboarding; LLM composes skills per task automatically after that.
