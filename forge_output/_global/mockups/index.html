⚠ Design contract — Forge UX is skeptical, not reassuring

Forge does not build confidence toward quick approval. Every screen surfaces what wasn't done, what wasn't checked, and what assumption was made silently.

Green metrics are starting points for the next question. "Approve" is tertiary; primary actions are "Ask challenger to verify X", "Add scenario I realized is missing", "Re-open with gap note".

If you see a "trustworthy"/"looks good"/"approve" framing in any mockup, that is a contract violation — flag it.

Project originSOW-driven (one doc)Initiative — sources added as KB over time
Knowledge inputFile upload only4 types: file, URL, folder, manual note — each with description + focus hints
Analysis / PlanningMonolithic buttonsTask types (ANALYSIS / PLANNING / DEVELOP) with AC, gates, scenarios, challenger

Tasks relationSequential listDAG per objective + cross-objective dependencies

AmbiguityOnly at analyze stepAny stage — bubbles back to owning objective as a "finish this" task

Acceptance criteriaFixed at planningUser can add AC / scenarios post-hoc. Forge auto-generates test if possible, else flags for manual review
